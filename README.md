# FSの仕様
このFSはパーティションとして得たいパーティションサイズを指定して逆算をしたうえでブロック、クラスタサイズを得る方法でブロック、クラスタサイズを決めています。  
クラスタにアクセスするにはツリー構造を使用しています。  
またこのFSは私の[自作OS](https://github.com/ban-himajin/MyOS-V2)で使用するために作成しています。  
**※このFSは開発途中なため冗長なコードやタイプミス、想定した動作をしないであろうコードが含まれる可能性があるため見つけた場合はお知らせください。**

## FSのデータ配置
1. スーパーブロック
2. bitmap
3. アドレスツリー
4. ディレクトリツリー
5. フリーIDツリー
6. バックアップリストツリー

[ ] アドレスツリーのアドレス幅の指定

## 計算
- 要求パーティションサイズ
- セクタ = デバイスから取得
- ブロック = nセクタ数
- クラスタ = nブロック数
- パーティションサイズ = nクラスタ数

# ファイルヘッダについて
ファイルヘッダは基本的にjsonを使い管理をする  
またjsonにすることで新たな拡張子でも対応が早くできるというのを狙っている  

## ファイルヘッダ時の規約
1. 必ずトップのキーは(.)を含む拡張子名にすること
2. .拡張子名 となった要素の中に入れ子として要素を入れること
3. 要素を入れることが可能。
    <br>次の要素が追加可能です。
    |キー|内容|デフォルト公開状態|
    |---|---|---|
    |**"Public"**|ファイルヘッダがもつべき情報として記述|public|
    |**"Magic"**|マジックナンバー情報|public|
    |**"Version"**|ファイルヘッダのバージョン情報|public|
    |**"DataOffset"**|実データがどこから始まるかの情報|public|
    |**"Width"**|画像データの横幅情報|public|
    |**"Height"**|画像データの縦幅情報|public|
    |**"ChecksumNum"**|チェックサムに使用する番号、またチェックサムの方法はChecksumIDを使い方法を識別|public|
    |**"Praivate"**|ファイルヘッダの設定(jsonのみに保存)|private|
    |**"ChecksumID"**|どのChecksumを使用するか(どのIDが何に対応するかは後述)|private|
4. 例
```json
{
    ".osys":{
        "Public":{
            "Magic":"OSYS",
            "Version":0,
            "DataOffset":0,
            "ChecksumNum":0
        },
        "Private":{
            "ChecksumID":1
        }
    }
}
```

- Unique Partition GUID条件:`8文字-4文字-4文字-4文字-12文字`で構成する

|文字数|8文字|4文字|4文字|4文字|12文字|
|-----|-----|-----|-----|-----|-----|
|エンディアン|リトル|リトル|リトル|ビック|ビック|

|8文字|4文字|4文字|4文字|12文字|
|-----|-----|-----|-----|-----|
|作成日|作成時分|フラグ(内容はフラグ内容参照)|シリアルナンバー|hmjnOS(ASCII Code)|
|xxxxxxxx|xxxx|xxxx|xxxx|0x686D6A6E4F53|

### フラグ内容
|bit数|意味|
|-----|-----|
|0|System/Root|
|1|User Visibl|
|2|Read Only|
|3|Critical|
|4|Bootable|
|5|Encrypted|
|6-11|*(Reserved)*|
|12-15|Category ID|


### 詳細
- **System/Root**(0bit)
    - 0.単なるデータ領域(ユーザーも使用可能)
    - 1.OSの起動・中核に使うデータを含む領域(SystemFileがある)(ユーザーも一部使える)
- **User Visible**(1bit)
    - 0.表示不可能パーティション
    - 1.表示可能パーティション
- **Read Only**(2bit)
    - 0.読み込み可、書き込み可
    - 1.読み込み可、書き込み不可
- **Critical**(3bit)
    - 0.フォーマット可
    - 1.フォーマット不可(保護領域)
- **Bootable**(4bit)
    - 0.ブート、起動対象ではない
    - 1.ブート、起動対象
- **Encrypted**(5bit)
    - 0.暗号化されていない
    - 1.暗号化されている
- **Reserved**(6-11bit)
    - 将来の拡張用(予約領域)
- **Category ID**(12-15bit)
    - 4bit分を使って使用用途をグループ分けする

### カテゴリID
|ID|カテゴリ名|内容|
|-----|-----|-----|
|0|Reserved|未定、または汎用的な領域|
|1|System Code|カーネル、ドライバ、システムバイナリ等々|
|2|User Data|ユーザーのドキュメント、設定、デスクトップファイルなど|
|3|Application|インストールされたソフトウェアの実行ファイル群|
|4|Swqp/Cache|メモリ一時退去領域や、フォントキャッシュなどの一時ファイル|
|5|Recovery|修理用OS、バックアップイメージ、インストローラー|
|6|Development|ソースコード、コンパイラ、デバッグ用(ログ含む)|
|E|Experimntal|試験的なファイルシステム屋、新しい機能のテスト用領域|
|F|External|ほかのOS(Windows/Linux)と共有するための互換領域|
